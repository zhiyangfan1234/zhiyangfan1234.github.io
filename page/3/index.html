B - Exactly N points

共有 $n$ 个物品，其中第 $i$ 个物品的权值是 $i$。现在想取出一些物品使得它们的权值和恰好是 $n$ 且最小化选取物品中权值的最大值，求出一种可能的方案。($1\le n\le10^7$)

首先考虑最最最简单的 $1\le n\le1000$ 的情况，显然可以直接背包 $\rm dp$ 然后随便记录一下路径，时间复杂度 $\mathcal{O}(n^2)$。接下来考虑沿着这个算法的思路继续思考。注意到物品的权值是比较特殊的，简单手玩一下就能发现，前 $i$ 个物品能组成的权值是 $1\sim \dfrac{i(i+1)}{2}$，因为要最小化最大权值，所以我们肯定要尽量用靠前的物品。所以我们可以列出：

解得 $x=\left\lceil\dfrac{-1+\sqrt{1+8n}}{2}\right\rceil$。我们只需要用前 $x$ 个物品就能满足条件了，构造方案可以直接尽量用权值较大的物品去减，最终时间复杂度 $\mathcal{O}(\sqrt{n})$。

C - Interpretation

在一个星球上有 $m$ 种语言，现在有 $n$ 个人，每个人会说 $k_i$ 种语言 $l_{i,1},l_{i,2},\cdot\cdot\cdot,l_{i,{k_i}}$。两个人能交流当且仅当以下条件满足：

  • 存在一种两个人都会讲的语言。
  • 存在一个两个人都能与他交流的人。

问这 $n$ 个人能否相互交流。($2\le n\le10^5,1\le k_i\le m\le10^5,\sum k_i\le 10^5,\forall i\ne j$，都有 $l_{g,i}\ne l_{g,j}$)

很容易 想到图论模型，把人向他所有会说的语言连边，最后看看所有点在不在同一个连通块中。会说同一种语言的人连通是显然的，而由于这种连通性可以传递，所以通过共同都能交流的人连通也是显然的。最终时间复杂度为 $\mathcal{O}(n\alpha(n))$。

D - Pair Cards

一共有 $n$ 张卡片，第 $i$ 张卡片上面写着 $x_i$。给出一个数 $m$，则两张卡片能配对当且仅当以下条件至少满足一个：

  • 两张卡片上面写的数相同。
  • 两张卡片上面写的数之和为 $m$ 的倍数。

求出最多能配多少对，注意一张卡片最多在一对中。($2\le n\le10^5,1\le m,x_i\le10^5$)

可以想到一个配对的思路是先配出所有两张卡片之和为 $m$ 的倍数（尽量不拆开相同的卡片）最后再合并一下两张卡片上面的数相同的对数。考虑为什么这样是对的，当不得不拆开相同的卡片时，这两张卡片如果只是自己之间配对，最多只能得到一对，但如果选择拿其中一个跟别人配对，另一个也有可能能跟另一个配对，最多可以得到两对，且至少一对。总之不亏，且有可能赚。题目中的倍数可以想到同余处理，开个桶记录一下 $m$ 的完全剩余系中每个值对应了多少张卡片，然后一一匹配即可。注意特判一下 $0$ 和 $\frac{m}{2}$ 的情况，因为它们是跟自己匹配的。为了匹配剩下的相同卡片，我们再记录一下每个余数对应着多少对相同的卡片，最后跟剩下多少张取个 $\min$ 即可（这样就可以做到尽量不拆开相同的卡片了）。时间复杂度 $\mathcal{O}(n+m)$。

E - Cookies

初始时做曲奇的速度为每秒 $1$ 个。可以任意次选择在任意时刻花 $a$ 秒吃掉当前做出的所有曲奇，如果吃掉了 $x$ 个曲奇，则做曲奇的速度变为每秒 $x$ 个。问做 $n$ 个曲奇至少需要多少秒，注意一秒不能拆开来算。($1\le n\le10^{12},0\le a\le10^{12}$)

在考虑正解之前，让我们先来看看 $1\le n\le 10^6$ 的部分分。我们设 $f_i$ 表示做出 恰好 $n$ 个曲奇的最小时间，我们从 $1$ 到 $n-1$ 枚举最后一轮的速度，如果当前的速度为 $i$，则显然有转移

而最后答案也不一定是 $f_n$，而是 $\min_{i=n}^{2n}\{f_i\}$。最终时间复杂度 $\mathcal{O}(n\log n)$，其中 $\log n$ 为枚举倍数调和级数级别的复杂度。

而 $1\le n\le10^{12}$ 的正解显然不能再用 $\rm dp$ 了，考虑数学推导。如果我们令曲奇数量为 $y$ 轴，时间为 $x$ 轴，则原题的过程可以转化为以下函数图像：

其中第一段直线的斜率为 $1$。则有 $c_1=t_1$，所以如果一共分为 $k$ 段，则做出的曲奇数应该为 $\prod_{i=1}^k t_i$ 块，用掉的时间为 $\sum_{i=1}^k t_i+a(k-1)$ 秒。所以问题就转化为了：

已知 $\prod_{i=1}^k t_i\ge n$，最小化 $\sum_{i=1}^k t_i+a(k-1)$ 的值。

注意到这个 $k$ 是 $\mathcal{O}(\log n)$ 级别的，显然我们可以直接枚举这个 $k$。$k$ 确定之后 $a(k-1)$ 也就变成常数了，而剩下来的形式就很像基本不等式了。考虑这个问题：

已知 $\prod_{i=1}^k t_i=n$，最小化 $\sum_{i=1}^k t_i$ 的值。

根据基本不等式，显然 $\sum_{i=1}^k t_i\ge k\sqrt[k]{\prod_{i=1}^k t_i}=k\sqrt[k]{n}$，等号成立当且仅当 $t_1=t_2=\cdot\cdot\cdot=t_k$，可以直接 $\mathcal{O}(1)$ 计算。但是原问题不是 $=$ 而是 $\ge$。不过区别并不大，注意到这个 $\ge$ 产生的原因其实是 $t_i$ 限定为整数，所以我们还是尽量让所有值相等，只不过让允许一些位置大 $1$ 使得恰好 $\ge n$，这样可以保证既 $\ge n$ 也尽量贴近基本不等式等号成立的条件。对于所有可能的 $k$，分别 $\mathcal{O}(1)$ 计算然后 $\min$ 即可，时间复杂度 $\mathcal{O}(\log n)$。

F - Road of the King

给出一张 $n$ 个点的有向图，定义一个序列 $c$ 是合法的当且仅当以下条件均满足：

  • $c$ 的长度为 $m$。
  • $\forall 1\le i\le m$，都有 $1\le c_i\le n$。
  • 如果在原图中加入 $m$ 条有向边 $c_{i-1}\rightarrow c_i(1\le i\le m)$，定义 $c_0=1$，则原图强联通。

求有多少种合法的序列，答案对 $10^9+7$ 取模。($2\le n\le300,1\le m\le300$)

考虑一个序列能构造出强联通的充要条件：

  1. 这个序列包含所有的点。
  2. 点 $1$ 能从点 $c_m$ 到达。

必要性显然，考虑如何证明充分性。对于条件 1，我们可以发现所有的点都能从 $1$ 到达，这样借 $1$ 为跳板，从所有都能到达 $c_m$。而根据条件 2，$c_m$ 能到达所有点，这样所有点就可以相互到达了，也就是原图强联通。

考虑 $\rm dp$，我们设 $f_{i,j,k}$ 表示当前序列长 $i$，已经用了 $j$ 个点，当前「强联通的状态」为 $k$ 的方案数。每次转移考虑把一个点加入序列中，这样就基本就可以描述所有需要的信息了，但问题是，「强联通的状态」到底该用什么表示呢？注意到如果我们加入的点是 $1$，则会产生一个强连通分量，如果我们加入的点能到达 $1$，则也会产生一个强联通分量，否则什么都不会改变。所以我们应该维护的是 包含 $1$ 的强联通分量大小。

接下来的 $\rm dp$ 转移就简单了，考虑三种情况：

  1. 加入一个没有在当前序列中出现过的点：$f_{i,j,k}=f_{i,j,k}+f_{i-1,j-1,k}\times (n-j+1)$
  2. 加入一个不在包含 $1$ 的强联通分量中的点：$f_{i,j,k}=f_{i,j,k}+f_{i-1,j,k}\times(j-k)$
  3. 加入一个在包含 $1$ 的强联通分量中的点：$f_{i,j,j}=f_{i,j,j}+f_{i-1,j,k}\times k$

初始值为 $f_{0,1,1}$，最终答案即为 $f_{m,n,n}$，$\mathcal{O}(n^2m)$ 转移即可。

#include <cstring>
#include <algorithm>
const int N = 2e5 + 10; int cst[N], f[N], tp, tot;
struct edge
{ 
    int u, v, w; 
    edge(int u = 0, int v = 0, int w = 0) : u(u), v(v), w(w) { }
}E[N << 1]; long long ans;
int getf(int x) { return x == f[x] ? x : f[x] = getf(f[x]); }
int main()
{
    memset(cst, 0x7f, sizeof (cst)); int n, q; scanf("%d%d", &n, &q); 
    for (int i = 1, A, B, C; i <= q; ++i) 
    {
        scanf("%d%d%d", &A, &B, &C); E[++tp] = edge(A, B, C);
        cst[A] = std::min(cst[A], C + 1); cst[B] = std::min(cst[B], C + 2);
    }
    for (int i = 0; i < n; ++i) cst[(i + 1) % n] = std::min(cst[(i + 1) % n], cst[i] + 2);
    for (int i = 0; i < n; ++i) cst[(i + 1) % n] = std::min(cst[(i + 1) % n], cst[i] + 2);
    for (int i = 0; i < n; ++i) E[++tp] = edge(i, (i + 1) % n, cst[i]), f[i] = i;
    std::sort(E + 1, E + tp + 1, [&](const edge& e1, const edge& e2) { return e1.w < e2.w; });
    for (int i = 1, tx, ty; i <= tp; ++i)
    {
        tx = getf(E[i].u), ty = getf(E[i].v);
        if (tx != ty) ans += E[i].w, f[tx] = ty, ++tot;
        if (tot == n - 1) break;
    }
    printf("%lld\n", ans); return 0;
}

H - Tokaido

有 $n$ 个方格从左到右编号为 $1\sim n$ 排成一列，初始时每一格上面有一个整数 $a_i$，Snuke 和 Rng 在按照以下的规则玩游戏：

  1. 两个人各执一枚棋子，Snuke 放在一号方格，Rng 放在二号方格。
  2. 棋子在另一个人左边的人移动，移动的目的地必须是它当前位置的右边且不为对手的位置。
  3. 重复执行2.步骤直到无法行动。
  4. 每个人的得分是他的棋子经过的格子上面的整数之和。

假设两个人都是足够聪明的，他们都想最大化自己比别人分数高的值。现在 $a_1\sim a_{n-1}$ 已经确定了，给出 $m$ 组询问 $x_i$，每组询问表示如果 $a_n=x_i$，则求出 Snuke 的分数 - Rng 的分数会是多少。($3\le n\le2\times10^5,0\le a_i\le10^6,\sum a_i\le10^6,1\le m\le2\times10^5,0\le x_i\le10^9$)

既然是博弈论，那么我们可以先考虑两个人怎么移动是最优的。注意到如果当前格子右边相邻的格子是空着的，则最优步数是棋子走到那里。原因显然，因为格子的权值非负，所以这样走不会损失分数，也不会损失自己的轮次。所以游戏的步骤（计分系统不变）如果改成这样是相同的：

  • 每次移动完棋子，玩家要把对手的棋子一格一格移动到他棋子的左边。
  • 每次移动完后两个棋子是相邻的。

B - Qualification simulator

给出 $n$ 个参赛者和两个常数 $A,B$，按排名递减分别考虑他们，其中 a 类选手能通过当且仅当已经确定通过的人数小于 $A+B$，b 类选手能通过当且仅当已经确定通过的人数小于 $A+B$ 且他在 b 类选手中排名 $B$ 或更高，c 类选手无法通过。现在按排名递减的顺序给出一个字符串，第 $i$ 位上表示该选手是哪一类的，排名第 $i$。求处每一位选手能否通过。($1\le n,A,B\le10^5$)

模拟一下就好了，需要稍微注意一点点初始值小细节之类的问题，但总归不是很难，时间复杂度 $\mathcal{O}(n)$。

C - Gr-idian MST

给出一个平面和 $W,H$，平面上的点 $(x,y)$ 均满足 $0\le x\le W,0\le y\le H$，在每个点 $(x,y),x,y\in \mathbf{Z}$ 上有个房子。能在所有满足 $x_1=x_2,|y_1-y_2|=1$ 或 $y_1=y_2,|x_1-x_2|=1$ 的点 $(x_1,y_1),(x_2,y_2)$ 之间修建一条路，其中对于所有 $j$，从 $(i,j)$ 到 $(i+1,j)$ 的道路价格均为 $p_i$，对于所有 $i$，从 $(i,j)$ 到 $(i,j+1)$ 的道路价格均为 $q_j$。现在想用最少的价格将所有房子连起来，求最少价格。($1\le W,H\le10^5,1\le p_i,q_j\le 10^8$)

首先题目和标题都按时我们这题是道最小生成树的题，但数据范围却告诉我们直接建图搞是会爆炸的，所以我们只能慢慢模拟。注意到这道题有很多边的价格是一样的，所以我们可以考虑依然用 $\rm Kruskal$ 算法的思想，但是要一次加好多边。在一般的 $\rm Kruskal$ 算法中，一条边不能加了当且仅当它加上去会形成环，而在这道题中，我们的边一加就是一列或一行，也要考虑环的问题。手玩一下样例可以发现（没发现就多造机几组数据玩一下），以行为例，当我们想加一行边时，加不上去的边个数等于已经加了多少列的边。因为一列的边已经全加上了（如果没有全加上，则说明已经快成环了，可以类比），则那一列对应的相邻点就不能同时加上行边，即有一条边加不上去，多个列都加过边的情况类似。这样我们把边的价格排序一下，从小到大依次考虑即可，时间复杂度 $\mathcal{O}((H+W)\log (H+W))$。

D - Greedy customers

商店前有一列共 $n$ 名顾客，每个顾客有 $a_i$ 元，商店老板每轮销售会展示一个商品，它的价格为 $P$，并从前向后询问，如果一个顾客满足 $a_i\ge P$，则询问到他时他就会买下这个商品，本轮结束。现在商店老板想进行尽可能多的轮次，并让顾客有钱回家，即所有的轮次进行完后所有顾客手中剩下的前大于 $0$，求出最多能进行多少轮。($1\le n\le10^5,1\le a_i\le 10^9$)

考虑一个显然的贪心策略，依次考虑每个人作为销售对象，每轮卖的东西价格尽量便宜，这样就能卖更多轮。但策略的问题是如果价格便宜到等于前面人手中剩下的钱，就会让前面的人剩下的前等于 $0$。所以价格应该满足的是大于前面所有人的最大值中尽量小的。但这样也有问题，因为我们想让前面的最大值尽量小，所以当我们面对比如卖 $3,4$ 元都行的局面（这种情况发生在对于当前销售对象，最多只能售卖一轮），就要卖尽量大的价格，这样才能让这个人对之后的决策影响最小。可能说的不是很清楚，具体见代码，时间复杂度 $\mathcal{O}(n)$。

E - Lexicographical disorder

给出 $n$ 个只含有小写字母的字符串，第 $i$ 个字符串标记为 $S_i$，且 $\forall i\ne j,S_i\ne S_j$。接下来有 $q$ 次询问，每次询问给出 $k$ 和 $p$，其中 $p$ 为字母 $\tt a\sim z$ 的排列，求出 $S_k$ 按照 $p_1<p_2<\cdot\cdot\cdot<p_{26}$ 的字典序排序后在原序列中的排名。($1\le n,q\le10^5,\sum |S_i|\le 4\times10^5$，保证输入合法)

注意到一个字符串 $s$ 小于一个字符串 $t$ 当且仅当下列条件成立其一：

  1. $s$ 是 $t$ 的前缀。
  2. 在 $s,t$ 第一次出现不同字符的位置上，$s$ 的字符小于 $t$ 的字符。

我们挨个讨论，对于条件 1，对于任意的字符串，不管给出的字典序是什么样的，前缀关系是不会变的，所以我们可以在回答询问之前先用 $\rm Trie$ 树求出这样的关系。

接下来看看条件 2，$s,t$ 第一次出现不同字符的位置也不依赖给出的字典序，所以我们依然可以先用 $\rm Trie$ 树求出这些位置，然后每次询问把 $S_k$ 和其他字符串比较这些地方，这样就能找到所有比 $S_k$ 小的字符串且不是 $S_k$ 前缀的数量。

但是这样朴素的比较回答询问的复杂度是 $\mathcal{O}(n)$ 的，无法接受。注意到小写字母总共只有 $26$ 个，也就是我们最多只需要比较 $26^2$ 对字符的大小关系，如果能提前预处理出来每对大小关系会贡献多少个比 $S_k$ 小的字符串，就可以做到以 $\mathcal{O}(|\sum|^2)$ 的复杂度回答询问，其中 $\sum$ 表示字符集。而上面所说的前缀数量，第一个不一样的位置和大小关系贡献的数量，都可以用 $\rm Trie$ 树在 $\mathcal{O}(|\sum|^2S)$ 的复杂度求出，其中 $S$ 表示所有字符串的长度和。

具体来讲，对于前缀的个数，我们构造出 $\rm Trie$ 树后在上面跑一遍 $\rm dfs$，用一个 s 记录一下从根节点到这个结点一共经过了多少个结束标志，则如果这个结点有 $k$ 的结束标记，那 $S_k$ 的前缀个数一共就有 s 个。注意到如果我们把 $S_k$ 本身看做自己的前缀，最后回答询问时就没必要再加一了。而对于第一个不一样的位置我们没必要求出来，因为我们只需要知道的是每对关系贡献的数量。对于每对关系贡献的数量，我们再跑一遍 $\rm dfs$，对于每个结点的两个儿子统计一下大小就可以了，具体实现见代码，总之最终可以做到 $\mathcal{O}(|\sum|^2S+q|\sum|^2)$ 的复杂度，足以通过本题。

总结

这把 AGC 比较有价值的就是最后一题提供的 $\rm Trie$ 树上找拐点和前缀来快速回答询问的思路了。之前做的弦论那道题是在 $\rm SAM$ 上面像在平衡树上回答第 $k$ 大一样的思路 跳来跳去，但这道题却因为时间复杂度不对不能这样做。但找到题目中什么量什么信息在询问中是不变的就可以把询问化到一个比较简单的形式，并用预处理提前找到那些不变的信息。

总之就是判看看有没有出现在 $\tt C$ 之后的 $\tt F$ 即可，时间复杂度 $\mathcal{O}(|s|)$。

B - K Cakes

现在有 $k$ 块蛋糕，蛋糕一共有 $n$ 种，每种有 $a_i$ 个。现在计划每天吃一块蛋糕，求最少有几天满足和上一天吃的蛋糕是同一种。($1\le k\le10^4,1\le n,a_i\le100,\sum a_i=k$)

一个小小小小的思维题。最优方案一定是每天都吃不一样的，直到剩下一种。考虑当前状况下最有可能留下的种类一定是 $a_i$ 最大的种类，而我们想要最小化这个种类剩下的数量，一个很显然的思路是每两天吃的是当前种类最多的和随便任意一种还剩的有的种类。最终答案就是剩下来的蛋糕数量减一，时间复杂度 $\mathcal{O}(kn)$。

C - Two Alpinists

有一个长为 $n$ 的正整数序列 $h$，给出两个序列 $T,A$ 来描述它。其中 $T_i=\max_{k=1}^i\{h_i\},A_i=\max_{k=i}^n\{h_i\}$。求出可能的合法 $h$ 序列数量对 $10^9+7$ 之后的结果。($1\le n\le 10^5,1\le T_i,A_i\le 10^9,\forall 1\le i<n, T_i\le T_{i+1},A_i\ge A_{i+1}$)

考虑先利用 $T$ 得出每个数的范围，再利用 $A$ 缩小范围或检查能否同时满足，得出每个数的最终范围之后乘起来就是答案。具体来讲，对于前缀最大值 $T$，它一定是单调不减的，当 $T_i>T_{i-1}$ 时，$h_i=T_i$ 一定成立，如果我们令 $T_0=0$，则这个对任意 $1\le i\le n$ 均成立，即这些位置的范围为 $l=r=T_i$。而对于满足 $T_i=T_{i-1}$ 的位置，对于 $h_i$ 的唯一限制就是不要超过当前的前缀最大值即可，这些位置的范围为 $l=1,r=T_i$。对于后缀最大值 $A$，处理是类似的，只不过要检查一下是否与 $T$ 已经确定过的范围冲突，如果冲突直接输出 $0$ 即可，否则则根据不等式相关知识合并即可。最终答案即为 $\prod_{i=1}^n (r_i-l_i+1)$，时间复杂度 $\mathcal{O}(n)$。

D - Friction

给出一个 $h$ 行 $w$ 列的表格，其中每一个格子 $(i,j)$ 都有一个用字母 a-z 表示的颜色 $c_{i,j}$。想用以下方法把这个表格拆除：

  • 选择 $w$ 列中的其中一列，把它往下移动一行，最后一行的格子消失。

每次操作需要一定的花费，具体来讲，每次操作的花费是满足以下条件的二元组 $(p,q)$ 个数：

  • 格子 $p,q$ 在移动这一列前水平相邻。

求出拆除所有表格所需要的最小花费。($1\le h\le300,2\le w\le 300$)

E - Encyclopedia of Permutations

首先考虑比较简单的前者。因为只涉及到 $\mathcal{P}_i$ 这一个位置，所以我们只需要分类讨论两种情况，注意下文中 $k$ 表示 $\mathcal{P}_i=0$ 的数量，$s$ 表示未被填入数字减一之和。

  1. $\mathcal{P}_i\ne 0$。则可以直接计算，即 $(\mathcal{P_i}-1)k!$。
  2. $\mathcal{P}_i=0$。可以稍微推一推式子然后直接计算。即 $s(k-1)!$，分配率乘进 $s$ 之后可以发现其实就是对于每一种可能的填入情况乘上剩余位随便放的方案数。

接下来我们考虑后者，涉及到 $\mathcal{P}_i,\mathcal{P}_j$ 这两个位置，所以我们需要分类讨论四种情况，注意下文中 $sum_i$ 表示大于 $s_i$ 的数之和，$cnt$ 表示考虑到的位置之前有多少空位置。

  1. $\mathcal{P}_i\ne 0,\mathcal{P}_j\ne 0$。也就是统计 $\mathcal{P}_i$ 这一个确定的位置前面有多少确定的比它小的数，直接开个树状数组记录一下即可。
  2. $\mathcal{P}_i\ne 0,\mathcal{P}_j=0$。也就是统计 $\mathcal{P}_i$ 这一个确定的位置前面有多少不确定的比它小的数。直接在没有填的位置选一个比 $s_i$ 小的数填一下就完事了，即 $pre\times(sum_1-sum_{s_i})\times(k-1)!$。
  3. $\mathcal{P}_i=0,\mathcal{P}_j\ne 0$。也就是统计 $\mathcal{P}_i$ 这个不确定的位置前面有多少确定的比它小的数。这个能前缀统计，对于每个确定的 $\mathcal{P}_j$，$pre=pre+sum_{\mathcal{P}_j}\times(k-1)!$，也就是考虑空位中比它大的元素，其他随便填。
  4. $\mathcal{P}_i=0,\mathcal{P}_j=0$。也就是统计 $\mathcal{P}_i$ 这个不确定的位置前面有多少不确定的比它小的数。也就是 $k$ 个数里面选俩，其他随便填，即 $cnt\times \binom{k}{2}\times(k-2)!$。
